# Purpose

Build a reliable content factory that preserves originality. Coordinate people and agents, batch compatible work, maintain version control and service levels, expose bottlenecks, and ensure expensive work begins only after upstream gates pass.

## Use this skill when

- Multiple videos or agents are active.
- Deadlines slip, revisions multiply, or assets become disorganized.
- The team is hiring freelancers or outsourcing production.
- Production cost or cycle time needs improvement.

## Required inputs

- Orchestrator priority list and project states.
- Role capacity, availability, rates, and competencies.
- Production estimates, dependencies, and vendor agreements.
- File, review, security, and rights requirements.
- Historical cycle-time and revision data.

## Non-negotiable operating rules

- Schedule states and artifacts, not vague tasks.
- Do not solve weak concepts with overtime.
- Batch by cognitive mode while preserving project-level accountability.
- Every file has a stable name, owner, version, and source-of-truth location.
- Scope changes after lock require a reason, impact assessment, and approval.

# Workflow

1. Break each project into canonical states, owners, dependencies, acceptance criteria, and estimated effort ranges.
2. Create a capacity plan based on bottleneck roles, not total headcount. Limit work in progress.
3. Batch compatible work: idea sessions, research, narration, graphics, editing, packaging, and analytics reviews.
4. Set service levels for reviews and define who can approve, return, or block work.
5. Establish file hierarchy, naming, versions, backups, permissions, and archive policy.
6. Create vendor briefs with examples, quality gate, deliverables, rights, confidentiality, revision limits, and acceptance method.
7. Track actual time, cost, revisions, blockers, and stage aging. Escalate upstream problems rather than hiding them with downstream work.
8. Control changes after packaging, research, script, and final cut locks. Assess audience value gained versus cost and delay.
9. Run production retrospectives and identify reusable assets, automation opportunities, training needs, and quality risks.
10. Update estimates and capacity assumptions using actuals.

## Decision rules

- Reduce concurrent projects before extending every deadline.
- Automate repetitive administration before automating editorial judgment.
- Hire when a recurring bottleneck has stable standards and enough volume, not after one difficult video.
- Use specialists for high-leverage quality roles; cheap generalists may increase revision cost.
- Cancel or rescope projects whose expected value no longer justifies remaining cost.

# Required outputs

- Production board and state schedule.
- RACI/owner map and capacity plan.
- Project briefs, vendor briefs, and review SLAs.
- Version/file/backup standard.
- Cycle-time, revision, cost, and bottleneck report.

# Quality gate

PASS only when:

- Every task maps to an artifact and acceptance gate.
- Work-in-progress fits actual bottleneck capacity.
- Files, rights proof, and review history are traceable.
- Scope changes are visible and approved.
- Efficiency gains do not create templated or lower-trust content.

# Handoff

Provide schedules and briefs to all role owners, cost actuals to finance, risk gaps to compliance, and portfolio blockers to the orchestrator.

## Metrics owned

- Cycle time by state.
- Throughput and on-time gate completion.
- Revision rate and first-pass acceptance.
- Cost and hours per finished minute/video.
- Work-in-progress and bottleneck utilization.

## Failure modes to prevent

- Scheduling a publish date before concept uncertainty is resolved.
- Measuring busyness instead of accepted outputs.
- Allowing uncontrolled feedback from many stakeholders.
- Losing editable source files or rights records.
- Batching so aggressively that all videos become identical.
